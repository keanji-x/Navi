# Navi Commands Reference

> Auto-generated by `navi init` — version $$VERSION$$

---

## 1. `navi list <FILE>` — Extract File Skeleton

Show all top-level definitions (functions, classes, interfaces, structs, `pub mod`, `use` declarations) in a file with bodies collapsed to `{ ... }`. Works well on re-export files like `mod.rs`.

```bash
navi list src/auth/user_service.ts
```

```
File: src/auth/user_service.ts
  12: export interface User { ... }
  25: export class UserService { ... }
  30:   public async login(req: LoginReq): Promise<Token> { ... }
  88:   private hashPassword(pwd: string): string { ... }
```

**When to use:** First step to understand a file's structure before diving deeper.

---

## 2. `navi jump <SYMBOL> [--path <DIR>] [--all]` — Jump to Definition

Find and display the complete source code of a symbol definition with ±3 lines of context.

| Flag | Description |
|------|-------------|
| `--path <DIR>` | Directory to search in (default: CWD) |
| `--all` | Show all definitions instead of just the first |

```bash
navi jump login --path src/
navi jump UserService --all
```

**When to use:** To read the full implementation of an unfamiliar symbol.

---

## 3. `navi refs <SYMBOL> [--path <DIR>]` — Find All References

Locate every file and line where a symbol is referenced.

| Flag | Description |
|------|-------------|
| `--path <DIR>` | Directory to search in (default: CWD) |

```bash
navi refs login --path src/
```

```
Found 3 references for 'login':
- src/api/routes.ts: 45 | const token = await userService.login(req.body);
- src/tests/auth.test.ts: 12 | const res = await service.login(mockUser);
- src/auth/user_service.ts: 30 | public async login(req: LoginReq): Promise<Token> {
```

**When to use:** Before modifying a function signature or renaming a symbol. Assess blast radius.

---

## 4. `navi read <FILE> <START-END>` — Read Line Range

Read a precise slice of a file by line numbers (1-indexed, inclusive). No AST parsing — just raw lines.

Supports both `-` and `:` as range separators:

```bash
navi read src/main.rs 10-25
navi read src/main.rs 10:25   # same result
```

**When to use:** When you already know the exact lines you need.

---

## 5. `navi tree [DIR] [--depth <N>]` — Recursive Directory Skeleton

Recursively walk a directory and run `list` on every supported source file, producing a full project skeleton.

| Flag | Description |
|------|-------------|
| `--depth <N>` | Max directory depth to recurse into (default: unlimited) |

```bash
navi tree src/
navi tree --depth 2    # only recurse 2 levels deep
navi tree              # defaults to CWD
```

**When to use:** To get a bird's-eye view of all definitions across a codebase or subdirectory.

---

## 6. `navi outline [DIR]` — Project Architecture Overview

Show a high-level project outline with package/crate organization and file counts.

```bash
navi outline
navi outline packages/
```

**When to use:** First command to run on an unfamiliar project to understand its structure.

---

## 7. `navi callers <SYMBOL> [--path <DIR>]` — Find Call-Sites

Find actual call-site usages of a symbol, excluding import statements and type annotations.

| Flag | Description |
|------|-------------|
| `--path <DIR>` | Directory to search in (default: CWD) |

```bash
navi callers handleRequest --path src/
```

**When to use:** To see where a function is actually *invoked* (not just imported).

---

## 8. `navi deps <FILE>` — File Dependencies

Show what a file imports and which other files import it (reverse dependencies).

```bash
navi deps src/auth/user_service.ts
```

**When to use:** To understand a file's dependency relationships before refactoring.

---

## 9. `navi types <SYMBOL> [--path <DIR>] [--depth <N>]` — Recursive Type Expansion

Look up a type/struct/interface definition and recursively expand all types it references.

| Flag | Description |
|------|-------------|
| `--path <DIR>` | Directory to search in (default: CWD) |
| `--depth <N>` | Max recursion depth (default: 1) |

```bash
navi types UserConfig --depth 2
```

**When to use:** To understand complex data shapes and their nested type dependencies.

---

## 10. `navi scope <FILE> <LINE>` — Show Enclosing Scope

Given a file and line number, show the enclosing function/method scope that contains that line.

```bash
navi scope src/main.rs 45
```

**When to use:** When you see a line number (e.g. from an error trace) and need to understand its context.

---

## 11. `navi diff <SYMBOL> [--path <DIR>]` — Symbol-Filtered Git Diff

Show the git diff filtered to only the hunks that touch a specific symbol's definition. First checks uncommitted changes (`git diff HEAD`), then falls back to staged-only changes (`git diff --cached`).

| Flag | Description |
|------|-------------|
| `--path <DIR>` | Directory to search in (default: CWD) |

```bash
navi diff login --path src/
```

**When to use:** To see exactly what changed in a specific function between commits.

---

## 12. `navi sg [ARGS...]` — ast-grep Passthrough

Forward arguments directly to the underlying `ast-grep` CLI (run, scan, test, etc.).

```bash
navi sg run --pattern 'console.log($$$)' --lang typescript
navi sg scan
```

**When to use:** For advanced AST pattern matching beyond Navi's built-in commands.

---

## 13. `navi init [DIR]` — Initialize Skill Documents

Create or update the Navi skill documents (`SKILL.md` and `COMMANDS.md`) in `.agent/skills/navi/`. Automatically detects version changes and updates in place.

```bash
navi init
navi init /path/to/project
```

**When to use:** Run once per project to teach AI agents about Navi's capabilities.
